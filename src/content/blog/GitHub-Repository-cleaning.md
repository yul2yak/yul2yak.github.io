GitHub 의 프로젝트들이 너무 많아졌다. 
학습하면서 만든 것들과 실제 사용하는 것들이 구분이 안되는게 일단 문제
README가 제대로 안되어 있으면 직접 코드를 보면서 기억해내야 한다

정리하기 위한 전략
- 일회성 리포지터리는 아카이빙하기
- README를 잘 적기
  - 목적과 현재 상태 정도가 있으면 좋을 것 같다
